Markdown是一种轻量级的标记语言,语法简单，使用它写文章，让我们能够更加专注于文章的内容，而不是排版。

从昨天初次接触Markdown就让我产生了浓厚的兴趣，所以整理了这份总结，希望能够帮助那些和我一样喜欢它的人。

Markdown编辑器

工欲善其事，必先利其器。在学习Markdown语言之前，你需要一个适合Markdown的编辑器。目前我使用的编辑器是sublime text2，一开始也用过网上推荐的haroopress，但感觉不是特别理想，在Hexo的博客框架下使用Markdown还是和其他有些不同，建议还是使用sublime。听说mac上的mou编辑器非常适合Markdown，但还没有机会尝试。
在sublime上使用Markdown，主要需要两个插件Markdown Preview和Markdown Extended。具体方法如下，可以参考这篇教程：sublime text 2 下的Markdown写作,不同是你需要安装两个插件，依样画葫芦即可。

Markdown的语法

Markdown的语法说明，可以参考献给写作者的 Markdown 新手指南和Markdown 语法说明 (简体中文版)这两篇文章，我就不再赘述了。主要讲一下我看来教程后遇到的几个问题。

Markdown摘要文字

为了防止在主页出现你的全部文章，一般我们会在Markdown的文章中截取一段，作为摘要文字，具体实现方法，就是在你的摘要文字下面另起一行加上<!-- more -->即可。

Markdown插入代码

在Hexo使用Markdown,与在其他地方略有不同，可以参考Hexo的官方文档
我的方法是使用三个` 加上c,具体在三个反引号后加什么，主要根据你的语言类型，具体有哪些类型可以参考这个网站。如果不知道就不加，或者加bash，不过这样就没有代码的高亮效果了。注意:在三个反引号和c之间需要有一个空格

注意
如果只是插入一小段行内代码，可以在行内代码前后各加一个反引号就可以达到这种效果,但如果行内代码里，也有反引号，就需要在代码前后各加两个反引号，比如我这段代码里有`反引号。但我遇到的问题是，如果代码里有超过三个反引号，hexo就会和下面的代码区段混淆，问题还未解决。
